# Authorization Overview

## Purpose
Этот раздел описывает модель авторизации, используемую в системе.
Авторизация определяет, КАКИЕ действия может выполнять аутентифицированный субъект
(ПОЛЬЗОВАТЕЛЬ или КЛИЕНТ) над ресурсами системы.

## Key Principles
- Аутентификация и авторизация строго разделены
- Авторизация оценивается на двух уровнях:
  - Coarse-grained (RBAC, OAuth2 scopes)
  - Fine-grained (доменные политики)
- Логика авторизации является частью домена, а не инфраструктуры

## Authorization Layers


| Уровень	| Ответственность |
|------|---------------|
| OAuth2 / Scopes	| Доступ на уровне API |
| Spring Security	| Контроль на уровне методов |
| AuthorizationService	| Доменная авторизация |
| AuthorizationPolicy	| Правила для конкретных ресурсов |

## Что делает авторизация
- Проверяет доступ к доменным ресурсам
- Обеспечивает соблюдение бизнес-ограничений
- Защищает границы tenant'ов (арендаторов)

## Что НЕ делает авторизация
- Аутентификацию пользователей
- Проверку токенов
- Логику навигации в пользовательском интерфейсе
