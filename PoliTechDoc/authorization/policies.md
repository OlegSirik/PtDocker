# Политики авторизации

## Назначение
AuthorizationPolicy определяет правила авторизации для конкретных ресурсов.

Политики отвечают на вопрос:
«Разрешено ли этому субъекту выполнять это действие над ЭТИМ ресурсом?»

## Примеры правил
- Только владелец может обновлять черновик
- Выпущенный полис доступен только для чтения
- Платеж может быть возвращен только в разрешенный период

## Характеристики политики
- Одна политика на тип ресурса
- Без логики инфраструктуры
- Полностью тестируемы изолированно

## Разрешение политики
Политики определяются по классу ресурса
и вызываются AuthorizationService.
