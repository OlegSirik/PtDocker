# Аутентификация

## Типы аутентификации
Система поддерживает два типа аутентификации:

| Тип | Описание |
|----|----------|
| USER | Запросы от имени пользователя |
| CLIENT | Технические или сервисные запросы |

## USER-аутентификация
- OAuth2 Authorization Code flow (+ PKCE)
- Требует участия пользователя
- Токен представляет пользователя

Используется для:
- UI-приложений
- Back-office
- Административных API

## CLIENT-аутентификация
- OAuth2 Client Credentials flow
- Нет пользовательского контекста
- Токен представляет сервис

Используется для:
- Фоновых задач
- Интеграций
- Внутренних сервисов
