# Обзор безопасности

## Назначение
В этом разделе описана архитектура безопасности системы,
включая механизмы аутентификации, работу с токенами
и интеграцию с внешним Identity Provider.

## Область ответственности
Безопасность включает:
- Аутентификацию (AuthN)
- Валидацию токенов
- Защиту транспортного уровня
- Границы доверия между системами

Авторизация (AuthZ) описывается в отдельном разделе документации.

## Ключевые принципы
- Соответствие OAuth2 / OpenID Connect
- Вынесенное управление идентификацией
- Stateless API
- Zero Trust между сервисами
