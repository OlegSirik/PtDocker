# Процесс расчета премии и создания договора

## Общая схема

```text
Quote Request
    ↓
DATA_VALIDATION
    ↓
PRODUCT_VALIDATION
    ↓
RATING
    ↓
POST_QUOTE_VALIDATION
    ↓
Quote Response

Policy Creation Request
    ↓
DATA_VALIDATION
    ↓
PRODUCT_VALIDATION
    ↓
RATING
    ↓
POST_QUOTE_VALIDATION
    ↓
PRE_SAVE_VALIDATION
    ↓
SAVE POLICY
    ↓
UNDERWRITING
    ↓
POST_SAVE_ACTIONS
```

---

# 1. DATA_VALIDATION

Проверка корректности входных данных.

Цель этапа — убедиться, что запрос может быть обработан системой.

## Проверяемые условия

### Обязательные поля

```text
Дата начала страхования
Дата рождения
Страна поездки
Страховая сумма
```

### Типы данных

```text
Дата
Число
Строка
Булево значение
```

### Допустимые значения

```text
Пол = M/F
Код страны существует
Код продукта существует
```

### Формат данных

```text
Email
Телефон
ИНН
Паспорт
```

При обнаружении ошибки процесс завершается.

---

# 2. PRODUCT_VALIDATION

Проверка соответствия запроса условиям страхового продукта.

На данном этапе выполняются продуктовые правила.

## Проверяемые условия

### Возрастные ограничения

Пример:

```text
Возраст должен быть от 18 до 75 лет.
```

### Ограничения по страховой сумме

Пример:

```text
Для возраста 65-75 лет максимальная страховая сумма составляет 5000.
```

### Ограничения по длительности страхования

Пример:

```text
Максимальная продолжительность поездки — 90 дней.
```

### Ограничения по территории страхования

Пример:

```text
Покрытие доступно только для стран Шенгенской зоны.
```

### Доступность покрытий

Пример:

```text
Спортивное покрытие доступно только для лиц младше 70 лет.
```

### Доступность франшиз

Пример:

```text
Франшиза 0 недоступна для водителей младше 25 лет.
```

### Доступность пакетов

Пример:

```text
Пакет PREMIUM доступен только при страховой сумме от 10000.
```

При нарушении любого правила возвращается бизнес-ошибка.

---

# 3. RATING

Расчет страховой премии.

На данном этапе выполняются только тарифные правила.

## Примеры

### Базовый тариф

```text
Премия = Базовый тариф × Коэффициент возраста
```

### Применение скидок

```text
Скидка за безубыточность
Промо-скидка
Партнерская скидка
```

### Применение надбавок

```text
Возрастная надбавка
Надбавка за рискованную территорию
```

Результатом этапа является рассчитанная премия.

---

# 4. POST_QUOTE_VALIDATION

Проверка результатов расчета.

Цель этапа — убедиться, что рассчитанные значения находятся в допустимых пределах.

## Проверяемые условия

### Минимальная премия

Пример:

```text
Премия не может быть меньше 10 EUR.
```

### Максимальная премия

Пример:

```text
Премия не может превышать 10000 EUR.
```

### Ограничения по комиссии

Пример:

```text
Комиссия агента не может превышать рассчитанную премию.
```

### Проверка технических лимитов

Пример:

```text
Премия должна быть положительной.
```

---

# 5. PRE_SAVE_VALIDATION

Дополнительные проверки перед сохранением договора.

Выполняются только при создании договора.

## Проверяемые условия

### Уникальность договора

```text
Номер договора не должен существовать.
```

### Проверка обязательных участников

```text
Страхователь указан.
Застрахованные лица указаны.
```

### Проверка платежных данных

```text
Корректность способа оплаты.
```

---

# 6. SAVE POLICY

Сохранение договора в базе данных.

На данном этапе:

```text
Создается договор.
Создаются покрытия.
Создаются участники.
Сохраняются расчетные результаты.
```

---

# 7. UNDERWRITING

Проверка договора на необходимость ручного согласования.

Этап выполняется после успешного создания договора.

## Примеры правил

### Высокая страховая сумма

```text
Страховая сумма более 100000.
```

### Рисковая территория

```text
Поездка в страну повышенного риска.
```

### Особые объекты страхования

```text
Автомобиль BMW.
Возраст автомобиля более 15 лет.
```

## Результат

```text
APPROVED
MANUAL_REVIEW
DECLINED
```

---

# 8. POST_SAVE_ACTIONS

Действия после сохранения договора.

## Примеры

### Генерация документов

```text
Полис
Сертификат
Счет
```

### Отправка уведомлений

```text
Email
SMS
Push
```

### Кросс-продажи

```text
Предложение дополнительных страховых продуктов.
Предложение партнерских сервисов.
```

### Интеграции

```text
CRM
DWH
BI
Платежные системы
```

---

# Классификация правил

## DATA_VALIDATION

Проверка структуры и корректности данных.

## PRODUCT_VALIDATION

Проверка соответствия условиям продукта.

Включает:

```text
Возрастные ограничения
Лимиты страховых сумм
Доступность покрытий
Доступность франшиз
Доступность пакетов
Территориальные ограничения
```

## RATING

Расчет премии и тарифных коэффициентов.

## POST_QUOTE_VALIDATION

Проверка результатов расчета.

## UNDERWRITING

Правила ручного согласования.

## CROSS_SELL

Правила рекомендаций дополнительных продуктов.
