# Создание тенанта

## 1. Вход в системный тенант (sys)
После установки системы выполните вход в системный тенант (sys) — это специальный тенант для администрирования платформы.

Параметр	Значение
URL	https://your-domain.com/sys/login
Тенант	sys
Логин	sys_admin
Пароль	полученный при установке

⚠️ Важно
* Тенант sys предназначен только для настройки платформы
* Работа в тенанте sys запрещена для бизнес-операций
* После создания бизнес-тенанта необходимо переключиться в него

## 2. Смена пароля системного администратора
При первом входе в систему обязательно смените пароль системного администратора.

Для этого перейдите на вкладку пользователей. Выберите единственного пользователя sys и смените ему пароль.

⚠️ Важно
* Не рекомендуется удалять пользователя sys, не заведя другого и не дав ему права sys_admin. 
Сделав это вам придется переустановить систему.

## 3. Создание нового тенанта
В меню администрирования создайте новый тенант для бизнес-потребностей.

Навигация
text
Администрирование → Тенанты → Создать тенант
Параметры нового тенанта
Поле	Описание	Пример

Код тенанта	Уникальный идентификатор (латиница)	mycompany
Название	Полное наименование	ООО "Моя Компания"

Статус	Активен / Неактивен	Активен
Тип аутентификации  Платформа поддерживает несколько основных типов аутентификации. При создании тенанта нужно указать, какой тип будет использоваться для него. 

После создания
Тенант будет создан и готов к настройке

Система автоматически создаст базовые роли и права доступа

## 4. Создание администратора тенанта

Для управления новым тенантом необходимо создать пользователя с ролью администратор тенанта.
Для это нужно создать пользователя, указав логин, пароль, ФИО, должность
Зайти в список тенант_админов и добавить у него запись с только что созданым логином.

## 5. Нужно перелогиниться

Сделать logout
Перейти по URL	https://your-domain.com/new_tenant/login
new_tenant in lower case !
ввести свой логин и пароль

Можно создавать других пользователей.
