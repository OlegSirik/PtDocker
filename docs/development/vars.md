Словарь данных vars

Платформа на вход методов quote, policy получает договор в формате json.
Теоретически он может быть любой.

На платформе есть словарь данных, несколько.
Для коробок используется ( с одним объектом страхования )

Словарь данных логически сгруппирован в древовидную структуру.
Словарь данных нужен для flatten преобразования вложенной структуры данных (массива, объекта, JSON) в плоскую (одноуровневую) структуру.
Формат дерева словаря данных не обязан совпадать со входным вектором.

Каждый узел содержит такую информацию.
{
            "varDataType": "STRING",
            "varCode": "ph_gender",
            "varName": "Пол",
            "varPath": "policyHolder.person.gender",
            "varType": "IN",
            "varValue": "",
            "varCdm": "policyHolder.person.gender",
            "varNr": "18",
            "id": 169,
            "parent_id": 341,
            "varList": "gender",
            "isSystem": false,
            "isDeleted": false,
            "name": "Пол"
},

где id  sequence number
parent_id - родитель в контексте словаря данных.

varCode - уникальный в рамках словаря данных код
varName - понятное описание
varType - тип узла. 
* IN - данные из входного вектора. set невозможен.
* OBJECT - тип узла объект, для группировки аттрибутов.
* CONST - константа. 
* VAR - переменная, значение которой можно менять

varDataType - тип данных "STRING", почти не используется.
varPath - путь, по которому можно получить значение из входного вектора. JsonPath
varValue - значение аттрибута. например для констант.
varList - название справочника, ключ хранится в этом атрибуте. При проверке входного вектора значение всегда проверяется по справочнику.

isSystem - флаг системного поля. Такое поле нельзя удалять из словаря.
isDeleted -  false признак удаленного атрибута. скрыто в UI 
isTarifFactor - признак, что это поле используется в касчете тарифа. в этом случае оно передается в калькулятор.
isOptional - для создания договора.

varCdm: "policyHolder.person.gender",  путь в дереве.

входной вектор IN , get - jsonPath, set - null
MAGIC - расчетные по водному вектору значения. тапа возрас, количество дней и т.д.
VAR + isSystem=true параметры, присваиваемые системой. Например номер договора.

CONST - только в калькуляторе, для констант расчета
