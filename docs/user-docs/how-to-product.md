# Как настроить страховой продукт

## 1. Настройка проуктовой группы

***Продуктовая группа (Product Family)*** — это базовая конфигурационная сущность платформы, объединяющая страховые продукты, относящиеся к одной линии бизнеса.

Продуктовая группа определяет общий словарь и набор элементов, которые используются всеми продуктами внутри группы.

В её состав входят:

* структура договора и его атрибуты
* данные страхователя и объектов страхования
* справочник доступных покрытий и рисков
* типы документов
* типовые тарифные таблицы
* формулы расчёта страховой премии
* шаблоны строк для формирования печатных форм

Использование продуктовой группы позволяет централизованно управлять общими справочниками и правилами, обеспечивая единообразие настроек и упрощая создание и сопровождение отдельных страховых продуктов.

### Схема договора

Платформа поставляется с максимально полной схемой договора страхования.
В процессе настройки продуктовой группы эту схему необходимо адаптировать под ваш продукт.

Схема договора соответствует структуре, используемой в API, но оптимизирована для настройки «коробочных» страховых продуктов.

При настройке рекомендуется:

* удалить лишние объекты страхования
* выбрать тип страхователя
* оставить только используемые атрибуты

Эта настройка в первую очередь влияет на удобство конфигурирования продукта и читаемость формул, проверок данных и печатных форм.

Настройка схемы договора не является обязательной. Однако без неё списки полей при настройке формул, проверок и печатных форм могут быть значительно больше.

### Покрытия

Справочник покрытий, которые могут использоваться при настройке страховых продуктов.

Для каждого покрытия задаются:

* код покрытия (используется в API)
* наименование
* список рисков

Список рисков в текущей версии платформы используется только в информационных целях.

### Файлы

Справочник типов файлов, которые могут формироваться для страхового продукта.

В большинстве случаев используется тип файла:

* policy — страховой полис

Иногда дополнительно используется:

* kid — информационный документ о продукте.

Код типа файла используется при запросе документа через API.

### Строки

Справочник шаблонных строк, используемых при формировании печатных форм.

Этот справочник является общим для всей продуктовой группы и позволяет повторно использовать одинаковые текстовые фрагменты в разных документах.

### Формулы

Справочник формул расчёта страховой премии.

Формулы добавляются в справочник автоматически после их создания при настройке конкретного продукта и могут быть повторно использованы в других продуктах той же продуктовой группы.

Предварительное создание формул без привязки к продукту не предусмотрено.

### Коэффициенты

Справочник тарифных коэффициентов и таблиц.

В продуктовой группе можно заранее настроить:

* типовые тарифные таблицы
* таблицы коэффициентов
* таблицы проверки минимальной и максимальной премии

Использование общего справочника коэффициентов позволяет ускорить настройку новых продуктов и обеспечить единообразие тарифных правил внутри линии бизнеса.



## Настройка продукта

При создании нового продукта необходимо:

* Выбрать продуктовую группу
* Указать код продукта (используется в API)
* Указать понятное название продукта

### Основные данные

Помимо кода и названия необходимо заполнить следующие параметры.

#### Страховая компания

Компания — владелец продукта.

#### Период до начала действия договора

Период между:

* датой выпуска договора
* датой начала его действия

Этот параметр используется:

* для автоматического заполнения даты начала договора (если значение фиксированное)
* для проверки введённых пользователем данных (если допускается несколько вариантов)

#### Срок действия договора

Используется:

* для автоматического расчёта даты окончания договора
* либо для проверки корректности введённой даты окончания.

#### Генератор номеров

Определяет алгоритм и шаблон формирования номера договора.

Генератор автоматически присваивает номер каждому новому договору.

### Правила и проверки
#### Правила договора

В текущей версии доступно одно правило:

Застрахованный = Страхователь

Если правило включено, при получении запроса на расчёт или создание договора платформа автоматически:

добавляет объект страхования
или
заменяет его на данные страхователя.

### Проверки предрасчёта

Проверки, выполняемые при ***предварительном расчёте договора.***

Они валидируют входные данные запроса до выполнения расчёта страховой премии.

### Проверки договора

Проверки, выполняемые при ***создании договора страхования.***

При этом также выполняются все проверки предрасчёта.

### Схема договора

При создании продукта используется копия схемы договора из продуктовой группы.
Эта схема может быть дополнительно кастомизирована для конкретного продукта.

В настройках схемы можно:

#### Указать тарифные факторы

***Тарифные факторы*** — это параметры договора, используемые при расчёте страховой премии.

После добавления тарифного фактора:

* атрибут становится доступным в калькуляторе
* поле становится обязательным при предрасчёте договора

#### Указать обязательные атрибуты

Можно задать атрибуты, обязательные для создания договора.
Для них автоматически будет добавлена проверка заполнения.

#### Ограничить допустимые значения

Для атрибутов можно задать список допустимых значений.

Например:

тип документа = ***PASSPORT***

Это также автоматически добавит соответствующую проверку данных.

### Пакеты покрытий

Пакеты покрытий позволяют задавать преднастроенные наборы страховых покрытий.

Если пакеты не настроены, используется пакет по умолчанию с кодом 0.

Для каждого пакета необходимо определить:

* список доступных покрытий
* обязательность каждого покрытия

Также в рамках пакета настраиваются:

* шаблоны печатных форм
* параметры калькулятора
### Шаблоны печати

Содержит копию шаблонов печатных форм из продуктовой группы.

Используется как справочник при настройке продукта.

### Test

Раздел Test позволяет:

* сохранять примеры API-запросов
* выполнять расчёт страховой премии
* создавать тестовые договоры

Это позволяет проверять корректность настройки продукта, не покидая платформу.
