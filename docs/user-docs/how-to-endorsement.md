# Допникик

## Доплата

Если в результате изменения договора возникла доплата, то на уровне продукта нужно указать стратегий по которой будет менятся график оплаты.

### Доступные стратегии
#### 1. SEPARATE (отдельная доплата)

Разница премии выносится в отдельный платеж.
Исходный график платежей не изменяется.

Пример

Исходный договор:

| № | Сумма | Статус    |
|---|-------|-----------|
| 1 | 250   | Оплачен   |
| 2 | 250   | Оплачен   |
| 3 | 250   | Не оплачен|
| 4 | 250   | Не оплачен|

Премия: 1000

После изменения:

Новая премия: 1600
Разница: +600

Результат:

| №  | Сумма          |
|----|----------------|
| 3  | 250            |
| 4  | 250            |
| +  | 600 (доплата)  |

Когда использовать
корпоративные продукты
страхование грузов
требуется прозрачность расчёта
важна отчётность и аудит

#### 2. REBILL (перерасчёт будущих платежей)

Будущие платежи пересчитываются с учётом новой премии.
Доплата за прошедший период выделяется отдельно.

Пример

Было:

| № | Сумма |
|---|-------|
| 3 | 250   |
| 4 | 250   |

Стало:

| №  | Сумма |
|----|-------|
| 3  | 250   |
| 4  | 250   |

Когда использовать
продукты с регулярными платежами
важна точность распределения по периодам

#### 3. SPREAD (распределение по будущим платежам)

Вся разница премии равномерно распределяется по оставшимся платежам.
Отдельный платеж не создаётся.

Пример

Было:

| №  | Сумма |
|----|-------|
| 3  | 250   |
| 4  | 250   |

Стало:

| №  | Сумма |
|----|-------|
| 3  | 450   |
| 4  | 450   |

Когда использовать
розничные продукты
e-commerce / embedded страхование
важна простота для клиента

#### 4. IMMEDIATE (в ближайший платёж)

Вся разница добавляется к ближайшему неоплаченному платежу.

Пример

| №  | Сумма |
|----|-------|
| 3  | 850   |
| 4  | 250   |

Когда использовать
короткие договоры
разовые корректировки
минимизация количества платежей

#### Исключение
Если вся премия оплачена на момент допника то возможен только вариант единоврменной доплаты
